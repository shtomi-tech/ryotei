# 旅程管理

年間計画を起点に、旅行の日程、持ち物、費用、カウントダウンを管理する静的アプリです。今日の予定表示、次の予定、地図リンク、端末内チェックリスト保存、印刷、オフラインキャッシュに対応しています。

年間ダッシュボードでは、各四半期に3泊4日以上の旅行を1回登録する目安と、1月〜12月の月別日程を確認できます。空の四半期カードからその四半期の旅行計画を追加でき、入力した計画はその端末の `localStorage` に保存されます。実際の旅程は従来どおり `data.js` に登録します。チェックリスト、現地支出、前回開いていた旅程も端末内に保存されます。

## 起動

起動時は年間ダッシュボードを表示します。日々の旅程は、画面上部の「今日の予定」「全日程」または旅行タブから開きます。

ポータルから開く場合は、次のコマンドを実行します。

```powershell
py C:\Users\shtom\dev\portal\launcher.py
```

静的ファイルとして直接確認する場合は、`index.html` をブラウザで開きます。

## 個人データ

実データは `data.js` に置き、Gitへは追加しません。公開用の雛形は `data.example.js` です。初回は `data.example.js` をコピーして `data.js` を作成してください。`data.js` は既存の起動方法を保つため、そのまま読み込まれます。

## データ注意

個人的な旅行情報を扱うため、公開前に `data.js` の内容を確認します。

## 年間計画のデータ形式（任意）

`data.js` には、必要に応じて `window.TRAVEL_PLAN` を追加できます。ダッシュボード上で追加した計画と併せて表示されます。日程が分からない四半期の実績は、`year` と `quarter` だけを指定して登録できます。その場合は「日程未登録」と表示され、3泊4日以上の判定対象にはなりません。

```js
window.TRAVEL_PLAN = {
  year: 2026,
  goal: { quarters: 4, minNights: 3, minDays: 4 },
  plans: [
    { id: "q4-onsen", title: "秋の温泉旅行", start: "2026-10-09", end: "2026-10-12", note: "" },
    { id: "q2-shikoku", title: "四国旅行", year: 2026, quarter: 2, note: "日程未登録" }
  ]
};
```
